## 已做之事

### 根本原因（Hang）

`src/Opencode/MessageTransform.fs` 在 `experimental.chat.messages.transform` hook 内调用 `readSessionTexts client sessionID directory` → 在 prompt 生命周期内自读同一 session → SSE 流 never terminates。

### 修复内容

**`src/Opencode/MessageTransform.fs`** — 改变 replay 文本来源：

```fsharp
// Before
let replayTexts () = readSessionTexts client sessionID directory

// After  
let replayTexts () = Promise.lift (extractTextsFromEncodedMessages messagesArr)
```

移除未用导入 `SessionIo`，加入 `MessageTransformCommon`，重命名 `client` → `_client`。

**`src/Opencode/NudgeEffect.fs`** — 压缩 anchor 去重由 `holder.Mutate` + `anchorIssuedSessions` 集合改为 `anchorAlreadyInHistory` 检查 + `anchorPromptIssued` 快照字段（前者破坏 Fable 元组解构）。

**`src/Kernel/Nudge/Types.fs`** — 回退至 e678aba（移除未用 `anchorIssuedSessions` 字段）。

**`tests/IntegrationEventTestsOpencode.fs`** — 扩展去验证：第二个 `session.idle` 触发后断言 `promptCalls.Count = 1`。

### 问题 1：Caps Prelude 未注入 LLM

**现象：** Mock LLM 收到的第二跳（真实 agent）body 仅含：

```json
{"role":"system","content":"You are opencode, an interactive CLI tool..."}
{"role":"user","content":"hi"}
```

不含 `# Kolmolgorov 宝典`，不含 `# 铁律`。`b.Contains "Kolmolgorov"` = false。

**已知事实：**

1. 插件已加载（write/read/todowrite 工具工作正常，mock LLM 第二跳能接收工具调用）
2. `experimental.chat.messages.transform` hook 已注册（`PluginCore.fs:92`）
3. 代码链路 `loadCapsForScope → findCapsFiles → buildCapsMessages → replaceArrayInPlace messagesArr` 在源码层完整
4. 单元测试 `IntegrationCapsSpecs.defaultPreludeWithoutCapsSpec`（`tests/IntegrationCapsSpecs.fs:58-73`）通过 → buildCapsMessages 纯函数正确
5. **变更前 replays 链路也是 `extractTextsFromEncodedMessages` 路径 — 这说明 caps 注入失败并非本次变更引入**

**可能根因方向：**

- OpenCode hook 协议期望 output 结构不符合（`tryGetMessagesArrayFromOutput` 解构失败，`messagesArr` 为 null → 整个 messagesTransform 静默退出）
- `replaceArrayInPlace` 回写后 OpenCode 未使用修改后的数组给 LLM（可能需要不同的 hook 键或写路径）
- `experimental.chat.system.transform` 仅设置 directory 不改 system text，caps 注入的 user message 前缀在 OpenCode 实际给 LLM 的 messages 数组之外

**需要执行：**

1. 在 `messagesTransform` 入口打 `console.log` 检测 `tryGetMessagesArrayFromOutput` 是否返回 `Some`
2. 对比 OpenCode demo/官方 hook 示例确认 `experimental.chat.messages.transform` 的 output 结构
3. 查 `chatHooks.fs` / `ChatMessageHook`（`chat.message` hook）是否在 messages transform 之前就截获消息 → 绕开 transform

### 问题 2：自定义工具未触发

executor / fuzzy_find / fuzzy_grep / investigator / coder / meditator / browser 均报告 `tool-called: false`。

write/read/todowrite 走 OpenCode 原生工具路径成功。自定义工具要么 schema 未注册成功，要么 tool.execute hook 出异常。
