# BAD.md —— 一次 F# 文件合并重构的惨痛教训

> 任务：把 ~100 个微文件合并成 ~30 个领域文件（namespace 收拢 + 相对 module）。
> 结局：脚本路线把人拖进死亡螺旋，每修一个坑都烧一次完整 build（7s+），
> 最后卡在一个"编译过、407/408 测试过、唯独 1 个 runtime 测试挂"的幽灵 bug 上。
>
> **结论先行：这种活就该用 coder 一个文件一个文件地手工合并、边合边 build。慢即是快。**

---

## 一、死亡螺旋（每一步都以为快好了，每一步都炸）

1. **依赖分析只看 `open`，漏了 FQN 引用**
   reader 子代理只报了 `open VibeFs.X.Y`，没报代码里海量的全限定引用
   （`Dyn.isNullish`、`MessageDecoder.messageInfo`、`CapsFormat` 实则依赖 `MessageDecoder`+`Dyn`）。
   → 分组出现隐藏环 → **394 个编译错误**。

2. **重新扫描真实依赖（open + FQN + 短名），修好分组**
   但撞上 F# 的潜规则：**同一 namespace 文件内，不能用全路径 `open VibeFs.Kernel.Sibling`
   引用同文件的兄弟模块**（"namespace Sibling is not defined"）。
   → 又是满屏 "namespace not defined"。

3. **加"兄弟短名化"（`VibeFs.Kernel.X` → `X`）**
   正则写错：`\bVibeFs\.` 前缀 + `re.escape(ns)`（ns 已含 `VibeFs.Kernel`）
   → 拼成 `\bVibeFs\.VibeFs\.Kernel\....`，**永远匹配不上**，一个都没替换。

4. **修好正则 → 编译终于过了 → 407/408 测试过**
   唯独 1 个 dedup 相关 runtime 测试挂。查了半天，根因是：
   **统一 +4 缩进腐蚀了多行三引号字符串 `"""..."""`**
   （F# 多行字符串的续行缩进是字符串内容的一部分，盲加 4 空格 = 改了字符串字面量）。
   而我的"逐行 un-indent 再 diff"校验被这个骗过了——它把字符串里的空格也一起减掉，
   所以 diff 全绿，实际产物是坏的。

---

## 二、根因：把 F# 当纯文本处理

F# 是**语义敏感**语言，机械字符操作必然踩雷：

- **多行字符串**（`@"..."` / `"""..."""`）：续行缩进属于字符串内容，不能动。
- **同 namespace 同文件兄弟模块**：只能用短名 `open Sibling`，全路径 `open VibeFs.N.Sibling` 编译不过。
- **模块依赖不止 `open`**：大量 FQN（`VibeFs.Kernel.Dyn.x`）和短名（`Dyn.x`，靠 `open VibeFs.Kernel` 命名空间）引用，
  只扫 `open` 会漏掉一大半边，分组就会成环。

"一个脚本一把梭搞定全部"的诱惑，让每个微妙 bug 都必须跑一次完整 build 才能暴露，
调试成本指数级爆炸。**看似快的路，最慢。**

---

## 三、教训（切记，切记）

### 慢即是快。用 coder 逐文件手工合并，边做边 build。

- **coder 理解 F# 语义**：缩进怎么加、字符串怎么保留、模块用全名还是短名、
  `open` 要不要改——它自然处理对。脚本只看字符，盲缩进必然腐蚀多行字符串。
- **小步快跑 + 立即验证**：合一个文件 → build 一次 → 绿了再合下一个。
  出错时二分范围天然就是"刚刚动过的那一个文件"，定位秒级。
  而不是"一次生成 30 个文件再回头 debug 一锅粥"。
- **机械脚本只配做无语义搬运**（挪目录、改名）。一旦碰到缩进敏感语言 + 字符串字面量 +
  模块解析规则，**立刻回到手工 / 语义工具**，不要逞强。

### 具体到这次任务，正确姿势本该是：

1. 用 reader 把真实依赖（open + FQN + 短名）扫全，画一张 DAG。
2. 按 DAG 定好分组与编译顺序（这一步脚本算没问题）。
3. **每个目标文件交给 coder**：让它读对应的几个微文件，在理解语义的前提下
   手工合并成 `namespace + 相对 module`，正确处理缩进、多行字符串、兄弟 `open`。
4. 合一个，build 一个，过了再下一个。

宁可慢，宁可笨，不要把 F# 当 sed 的输入。

---

## 四、一句话

> 脚本让你在"快好了"的幻觉里反复返工；手工 + coder 让你在每一步都脚踏实地。
> **慢即是快。**
