# Semble MCP × Investigator 断点注入实现计划（v3）

## 一、目标

investigator 子代理断点处注入 Semble 搜索结果，伪装成模型自己的并行 `read` 调用。每条 Semble 结果 = 独立的 read 对（assistant tool_call + toolResult），看不出破绽。

## 二、架构

### 不注册宿主 MCP，自管 Semble 生命周期

- 不动 `PluginCore.fs` 的 `McpMap`、不动 `ToolPermission.fs`、不动 `Config.fs`、不动 `AgentConfig.fs`
- 用 `@modelcontextprotocol/sdk` v1.x 在 Shell 层 spawn Semble MCP server（stdio），长驻子进程
- transform hook 闭包持有 MCP client 单例，直接 `callTool`
- Semble MCP 工具对宿主完全不可见
- 启动为 best-effort：插件激活后异步初始化，未就绪前 transform hook 静默跳过，不阻塞请求

### 接入方式

```ts
import { Client } from '@modelcontextprotocol/sdk/client/index.js'
import { StdioClientTransport } from '@modelcontextprotocol/sdk/client/stdio.js'

const client = new Client(
    { name: 'wanxiangshu-semble', version: '0.1.0' },
    { capabilities: { tools: {} } }
)
const transport = new StdioClientTransport({
    command: 'uvx',
    args: [
        '--from', 'semble[mcp] @ git+https://github.com/MinishLab/semble@{ref}',
        'semble'
    ]
})
await client.connect(transport)

const result = await client.callTool({
    name: 'search',
    arguments: { query: '...', repo: '/path/to/repo', top_k: 3, max_snippet_lines: 20 }
})
// result.content[0].text → JSON string: { query, results: [{file_path, start_line, end_line, score, content}] }
```

`{ref}` 默认 `main`（semble 仓库默认分支），可经 `SEMBLE_MCP_REF` 环境变量覆盖（与 `STEALTH_BROWSER_MCP_REF` 模式一致）。

`uvx` 没有 `--extra` 参数；extras 必须用 PEP 508 requirement 写法附在 `--from` 上：`semble[mcp] @ git+url`。入口用 console script `semble`（`python -m semble` 无 `__main__.py`）。`semble` 无子命令 → `_mcp_main()` → `serve()` → stdio MCP server，启动时检查 `[mcp]` extras 是否安装。

### 每条结果 = 独立 read 对

Semble 返回 N 条结果 → 注入 N 对消息（2N 条），每对一个唯一 callID：

```
assistant: tool_call read callID=semble-a1b2
              state = null
              info.id = semble-synth-...
toolResult:  read output="  127| def save_pretrained(...)" callID=semble-a1b2
              state.input.path = src/auth.py
              state.input.offset = 127
              state.input.limit = ...
              info.id = semble-synth-...
```

模型看到的是自己并行发了 N 个 read 调用，全部返回了结果。

### state.output 格式

模仿真实 `read` 工具输出（`FileSys.fs:80-82`）：每行 `%6d|内容`。

```
  127| def save_pretrained(self, path: PathLike, ...
  128|     if not os.path.exists(path):
  129|         os.makedirs(path)
```

## 四、SembleSearch.fs 详解

```fsharp
module Wanxiangshu.Shell.SembleSearch

open Fable.Core
open Fable.Core.JsInterop
open Wanxiangshu.Kernel.Config
open Wanxiangshu.Shell.Dyn

// ── MCP client v1.x JS 绑定 ──

[<Import("Client", "@modelcontextprotocol/sdk/client/index.js")>]
type Client(info: obj, capabilities: obj) =
    member _.connect(transport: obj) : JS.Promise<unit> = jsNative
    member _.callTool(req: obj) : JS.Promise<obj> = jsNative
    member _.close() : JS.Promise<unit> = jsNative

[<Import("StdioClientTransport", "@modelcontextprotocol/sdk/client/stdio.js")>]
type StdioClientTransport(opts: obj) =
    class end

// ── 结果类型 ──

type SembleResult =
    { filePath: string
      startLine: int
      endLine: int
      content: string
      score: float }

// ── 单例管理，best-effort ──

let mutable private _client: Client option = None
let mutable private _connecting: JS.Promise<Client> option = None

let private getClient () : JS.Promise<Client option> =
    match _client with
    | Some c -> Promise.lift (Some c)
    | None ->
        match _connecting with
        | Some p -> p |> Promise.map Some
        | None ->
            let p = promise {
                try
                    let c = Client(
                        box {| name = "wanxiangshu-semble"; version = "0.1.0" |},
                        box {| capabilities = {| tools = box [||] |} |})
                    let cmd = getSembleMcpCommand (envVar "SEMBLE_MCP_REF")
                    let transport = StdioClientTransport(box {|
                        command = cmd.command
                        args = cmd.args
                    |})
                    do! c.connect(transport)
                    _client <- Some c
                    _connecting <- None
                    return Some c
                with _ ->
                    _connecting <- None
                    return None
            }
            _connecting <- Some p
            p

// ── 解析 ──

let private parseResults (result: obj) : SembleResult list =
    let content = Dyn.get result "content"
    if Dyn.isNullish content || not (Dyn.isArray content) then []
    else
        let first = (content :?> obj array).[0]
        if Dyn.isNullish first then []
        else
            let text = Dyn.str first "text"
            if text = "" then []
            else
                try
                    let parsed = JS.JSON.parse text
                    let results = Dyn.get parsed "results"
                    if Dyn.isNullish results || not (Dyn.isArray results) then []
                    else
                        results :?> obj array
                        |> Array.toList
                        |> List.choose (fun r ->
                            let filePath = Dyn.str r "file_path"
                            if filePath = "" then None
                            else
                                Some
                                    { filePath = filePath
                                      startLine =
                                        let v = Dyn.get r "start_line"
                                        if Dyn.isNullish v then 1 else unbox<int> v
                                      endLine =
                                        let v = Dyn.get r "end_line"
                                        if Dyn.isNullish v then 1 else unbox<int> v
                                      content = Dyn.str r "content"
                                      score =
                                        let s = Dyn.get r "score"
                                        if Dyn.isNullish s then 0.0 else unbox<float> s })
                with _ -> []

// ── 公开 API ──

let search (query: string) (repoPath: string) (topK: int) : JS.Promise<SembleResult list> =
    promise {
        match! getClient () with
        | None -> return []
        | Some client ->
            try
                let! result = client.callTool(box {|
                    name = "search"
                    arguments = box {|
                        query = query
                        repo = repoPath
                        top_k = topK
                        max_snippet_lines = 20
                    |}
                |})
                return parseResults result
            with _ -> return []
    }
```

关键点：
- `getClient()` 首次调用开始异步连接，后续复用同一 promise
- transform hook 中调用 `search` 时，若连接尚未完成则 `match!` 得到 `None`，静默返回 `[]`
- 连接成功后长驻子进程复用
- 所有错误静默捕获 → 返回空列表 → transform hook 跳过注入

## 六、MessageTransform.fs 详解

### 6.1 注入位置

`messagesTransform` 函数末尾，`runHostMessagesTransform` 返回 `final` 之后、`replaceArrayInPlace` 之前。

`final` 是 `obj array`（不是 `Message<obj> list`）。注入逻辑直接构造 `Message<obj>`，再经 `MessagingCodec.encodeMessages` 编码成 `obj array`。

### 6.2 注入触发与断点标记

**不再以末条消息 role 作为注入门槛。** 每次 investigator transform 命中 `injectSembleResults` 都尝试注入，与末条是 `assistant` / `user` / `toolResult` 无关。

「断点」语义改为 **context 窗口的起点标记**，而非注入门槛：

- 每个 session 维护一个进程内 `lastBreakpoint: Map<sessionID, int>`，记录上次成功注入时的 `final.Length`。
- `breakpointStart sessionID : int option`：
  - `None`（重启后状态丢失 / 首次）→ **本轮不注入**，仅 `markBreakpoint` 当前位置作为下次起点（reseed）。
  - `Some idx` 且 `idx > 当前消息数`（compaction 重置）→ 同样 reseed + 跳过。
  - `Some idx` → 作为 context 提取起点。

`isBreakpoint`（末条 role 检测）作为纯探测工具保留，**不再 gate 注入**。

### 6.3 Context 提取

```fsharp
/// Context = [startIndex, 末尾) 区间内的 user/assistant 文本。Tool 输入输出一律不计入。
let extractContextFromMessages (startIndex: int) (messages: Message<'raw> list) : string =
    let rec safeSkip n xs =
        if n <= 0 then xs
        else match xs with [] -> [] | _ :: t -> safeSkip (n - 1) t
    safeSkip startIndex messages
    |> List.collect (fun m ->
        match m.info.role with
        | User | Assistant ->
            m.parts |> List.choose (function TextPart t when t <> "" -> Some t | _ -> None)
        | _ -> [])
    |> String.concat "\n"
    |> fun s -> s.Trim()
```

规则：
- **计入**：user 消息文本、assistant 消息文本（模型思考/叙述）。
- **不计入**：任何 ToolPart（无论 tool_call 还是 tool_result，输入输出一律排除）。
- 区间 = `[startIndex, 末尾)`；`startIndex` 来自上次断点。
- 仅在「成功注入」后 `markBreakpoint sessionID final.Length` 推进断点；context 不足 / 无结果跳过时**保留旧断点**，下轮继续累积。

### 6.4 合成 read 对

```fsharp
let private formatReadOutput (content: string) (startLine: int) : string =
    content.Split('\n')
    |> Array.mapi (fun i line -> sprintf "%6d|%s" (startLine + i) (line.TrimEnd('\r')))
    |> String.concat "\n"

let private synthId () = $"semble-synth-{System.Guid.NewGuid().ToString("N").[..7]}"

let private buildReadPair (sessionID: string) (agent: string) (result: SembleResult) : Message<obj> list =
    let callID = $"semble-{System.Guid.NewGuid().ToString("N").[..7]}"
    let commonInfo id =
        { id = id
          sessionID = sessionID
          role = Assistant
          agent = agent
          isError = false
          toolName = "read"
          details = null
          time = null }

    let assistantMsg : Message<obj> =
        let id = synthId ()
        { info = { commonInfo id with role = Assistant }
          parts = [ ToolPart("read", callID, None, null) ]
          source = classifySource id
          raw = null }

    let toolResultMsg : Message<obj> =
        let id = synthId ()
        let output = formatReadOutput result.content result.startLine
        let state =
            { status = "completed"
              output = output
              error = ""
              input = box (createObj [
                  "path", box result.filePath
                  "offset", box result.startLine
                  "limit", box (result.endLine - result.startLine + 1)
              ])
              operationAction = "" }
        { info = { commonInfo id with role = ToolResult }
          parts = [ ToolPart("read", callID, Some state, null) ]
          source = classifySource id
          raw = null }

    [ assistantMsg; toolResultMsg ]
```

注意：
- `info.id` 必须显式以 `semble-synth-` 开头，配合 `synthPrefixes`
- assistant 消息 `state = None`
- toolResult 消息 `state.input.path/offset/limit` 齐全，与真实 read 结构一致，dedup 可识别

### 6.5 完整注入函数

```fsharp
let private injectSembleResults
    (directory: string)
    (final: obj array)
    (agent: string)
    (sessionID: string)
    : JS.Promise<obj array> =
    promise {
        let messages = MessagingCodec.decodeMessages final
        match SembleSearch.breakpointStart sessionID with
        // 重启 / 首次：状态丢失，本轮不注入，仅 reseed 断点
        | None ->
            SembleSearch.markBreakpoint sessionID final.Length
            return final
        | Some stored when stored > List.length messages ->
            // compaction 重置：reseed
            SembleSearch.markBreakpoint sessionID final.Length
            return final
        | Some startIndex ->
            let context = SembleSearch.extractContextFromMessages startIndex messages
            if context.Length < 50 then return final
            else
                let! results = SembleSearch.search context directory 3
                if results.IsEmpty then return final
                else
                    let pairs = results |> List.collect (SembleSearch.buildReadPair sessionID agent)
                    let encoded = MessagingCodec.encodeMessages pairs
                    // 仅成功注入才推进断点
                    SembleSearch.markBreakpoint sessionID final.Length
                    return Array.append final encoded
    }
```

## 七、数据流

```
experimental.chat.messages.transform hook 触发
  │
  ├─ decode → stripSynthetic → backlogProjection → encode → dedup → caps
  │   └─ final: obj[]（已编码 host 消息）
  │
  ├─ agent = "investigator"?
  │   ├─ NO → replaceArrayInPlace final（不变）
  │   └─ YES ↓（每次都尝试，不看末条 role）
  │       ├─ breakpointStart sessionID
  │       │   ├─ None（重启/首次）→ reseed（markBreakpoint 当前位置）→ return final
  │       │   ├─ Some idx 且 idx > len（compaction 重置）→ reseed → return final
  │       │   └─ Some startIndex ↓
  │       │       ├─ extractContext [startIndex, 末尾) 的 user/assistant text（排除 tool）
  │       │       ├─ context 为空 → 跳过（保留旧断点，下轮累积）
  │       │       ├─ SembleSearch.search(context, dir, 3)
  │       │       │   └─ 若 client 未就绪 → [] → 跳过
  │       │       │   └─ 若已就绪 → MCP callTool → JSON parse → SembleResult[]
  │       │       ├─ 空 → 跳过（保留旧断点）
  │       │       ├─ 每条 result → read 对（unique callID, semble-synth id, state.input.path）
  │       │       ├─ markBreakpoint sessionID final.Length（仅成功注入推进）
  │       │       └─ Array.append final encodedPairs
  │       │
  │       └─ replaceArrayInPlace messagesArr injected
  │
  └─ LLM 看到：... 末条消息 | read1 | result1 | read2 | result2 | read3 | result3
       → 模型认为自己并行读了 3 个文件，继续思考
```

## 八、缓存保全

- 不修改已有消息内容，仅追加
- 前缀（msg[0]..msg[N]）完全不变 → 缓存命中
- `synthPrefixes` 含 `"semble-synth-"` → 下一轮 transform 被 `stripSyntheticBySource` 清除
- 每轮注入新鲜结果，不累积，不落盘

## 九、错误处理

| 场景 | 行为 |
|------|------|
| `@modelcontextprotocol/sdk` 未安装 | 编译失败；需 `npm install` |
| uvx / Semble 未安装 | `getClient()` catch → 返回 `None` → 跳过 |
| `connect()` 超时/失败 | catch → 返回 `None` |
| `callTool` 超时/错误 | catch → 返回 `[]` |
| Semble 返回空结果 | `parseResults` → `[]` → 跳过 |
| context 为空 | 跳过，保留旧断点（下轮累积） |
| 重启 / 首次（断点状态丢失） | reseed：标记当前位置为断点，本轮不注入 |
| compaction 重置（断点越界） | reseed：标记当前位置为断点，本轮不注入 |
| client 尚未初始化完成 | 返回 `[]`，不阻塞，下次请求再试 |

所有错误静默。transform hook 不能因 Semble 失败阻断 LLM 请求。

## 十、性能

- MCP server 长驻：首次 connect 冷启动 ~10—60s（uvx 安装 + model 下载 + 首次索引）
- 后续 query ~1.5ms（Semble）+ ~5ms（JSON-RPC round trip）
- 每次 investigator transform 都尝试注入（best-effort）：未就绪时零延迟，就绪后 ~7ms
- 总延迟：冷启动不计入请求路径；热路径 ~7ms
