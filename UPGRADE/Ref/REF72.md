# REF72: Contextual 策略池演化算法

## 1. 策略池的基本规则

- 池大小：12-15 个正交策略方向
- 每个方案必须有**不同的最终答案或结论**
- 数值问题：每个方案必须产生不同的数值
- 置信度动态更新

## 2. 置信度更新协议

```typescript
// 策略池代理接受批判反馈后必须大幅度更新置信度
// 旧的 0.9 置信度方案被批判发现缺陷 → 降到 0.5 或更低
// 旧的 0.3 置信度方案被验证 → 升到 0.6 或更高
```

## 3. 退出条件检测

```typescript
// 策略池代理必须在内部维护计数器:
let consecutiveZeroFlaws = 0

if (critiqueFoundNoFlaws) {
    consecutiveZeroFlaws++
    if (consecutiveZeroFlaws >= 3) {
        output = '<<<Exit>>>'
        // 停止整个 Contextual 流程
    }
} else {
    consecutiveZeroFlaws = 0
}
```

## 4. 策略方向生成协议

策略池代理生成新策略时必须探索**最小可能值**：
```
你不能证明 x 是最小值。如果你认为自己证明了，那正是最大的局限——
你会把其他所有方案构造在 >= x 的范围里。
这限制了对真实最小值的探索。
```

## 5. 策略池与各代理的交互

| 代理 | 交互方式 |
|------|----------|
| 主生成器 | 接收完整池，从中选择或合成 |
| 迭代代理 | 为池提供批判依据 |
| 记忆代理 | 提供历史模式 |
| 策略池代理 | 接收反馈后更新池 |
