# REF73: Deepthink 同步屏障机制详解

## 1. 同步屏障的必要性

Deepthink EDFS 中有多个并行执行的代理组，某些阶段必须在所有并行任务完成后才能推进。

## 2. 同步点

```
迭代开始 → 修正器(并行) → [等待所有修正完成]
  → 批判器(并行) → [等待所有批判完成]
  → 方案池(并行) + 假设心跳(并行) → [等待两者完成: 偶数迭代]
  → PQF(并行) + 记忆(并行) → [等待两者完成: 每5轮]
  → 策略替换 → 迭代结束
```

## 3. 快照一致性

同步屏障确保：
```
修正开始时:
  - 分支 A 看到分支 B 的上次迭代结束状态
  - 分支 B 看到分支 A 的上次迭代结束状态
  - 不是分支 A 开始修正时，分支 B 还在修正中

方案池开始时:
  - 池 A 看到分支 B 的最新池（上轮）
  - 不是池 B 正在同时生成的内容
```

## 4. 实现方式

```typescript
// 使用 Promise.all 等待所有并行任务
await Promise.allSettled(activeStrategies.flatMap(strategy => {
    return subs.map(async subStrategy => {
        // 每个分支的执行
    })
}))
```

## 5. 奇数/偶数迭代差异

| 迭代 | 等待条件 |
|------|----------|
| 奇数（1,3,5,7,9） | 方案池完成 |
| 偶数（2,4,6,8,10） | 方案池完成 + 假设心跳完成 |
