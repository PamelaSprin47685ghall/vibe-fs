# REF96: Contextual Prompts 详解——策略池代理

## 1. 角色定位

策略池代理（也叫做"发散探索者"）的目标是：
- 维护 12-15 个正交策略方向
- 每个方向必须产生不同的最终结论
- 数值问题：每个方案必须产生不同的数值

## 2. 核心操作原则

```typescript
// 保证背书的多样性
- 绝对承诺多样性：没有两个方案共享相同最终答案
- 策略正交性 > 表面变体
- 动态置信度校准（0.0-1.0）
- 反教条主义（最低置信度的方案可能是关键）
- 持续从反馈中学习
```

## 3. 置信度更新协议

```typescript
// 必须根据批判大幅调整置信度
// 0.9 → 发现缺陷 → 降到 0.5 或更低
// 0.3 → 被验证 → 升到 0.6 或更高
```

## 4. 一致性检查规则

```typescript
// 每个方案必须通过:
- 逻辑步骤是否连贯？
- 是否解决了实际问题（或更简单变体）？
- 推理是否确实导向了声明的结论？
- 假设是否足够明确以便评估？
```

## 5. 退出机制

```typescript
// 策略池代理跟踪批判是否连续 3 次未发现缺陷
if (critique.foundNoFlaws) {
    counter++
    if (counter >= 3) {
        output = '<<<Exit>>>'
    }
} else {
    counter = 0
}
```
