# REF57: Deepthink 迭代编号规则与同步屏障

## 1. 四种编号系统

| 编号 | 含义 | 变更时机 |
|------|------|----------|
| 策略 ID | 稳定槽位（如 main1） | PQF 替换时不变 |
| 分支版本 | 分支演化版本 | PQF 替换时 +1 |
| 分支本地迭代 | 当前分支的输出计数 | 每次修正后 +1 |
| 全局迭代 | 所有分支共享的周期数 | 每次同步后 +1 |

## 2. 同步屏障

### 奇数迭代完成条件
所有方案池代理的输出就绪。

### 偶数迭代完成条件
所有方案池代理的输出 + 所有假设测试代理的输出就绪。

## 3. 迭代调度时序

```
全局迭代 1:
  初始化执行 → 批判 → [屏障: 等待所有分支]
  方案池 → [屏障: 所有池就绪]

全局迭代 2:
  修正 → 批判 → [屏障: 等待所有分支]
  方案池 || 假设心跳 → [屏障: 池+假设都就绪]

...

全局迭代 5:
  修正 → 批判 → [屏障: 等待所有分支]
  方案池 || 假设心跳(如果偶数) → [屏障]
  PQF + 记忆(并行) → [屏障: 都就绪]
  
全局迭代 6:
  (如果 PQF 替换了策略，新分支从这里开始)
  ...
```

## 4. 分支替换后的编号对齐

当 PQF 在全局迭代 5 替换一个策略：
- 新分支的**分支本地迭代**从 1 重新开始
- **全局迭代**仍然为 6（与其他分支对齐）
- **分支版本** +1

## 5. 记忆触发条件

绑定到分支本地迭代（而非全局迭代）：
```typescript
if (runtime.history.length - runtime.lastMemoryHistoryCount >= MEMORY_INTERVAL) {
    // 触发记忆
}
```
