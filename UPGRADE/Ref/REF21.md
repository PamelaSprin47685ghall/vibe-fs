# REF21: Deepthink 模式——结构化解决方案池 (Structured Solution Pool)

## 1. 角色定位

解决方案池代理是 EDFS 内部的重复广度优先搜索组件。它不产生权威最终答案，而是在修正代理的深度优先路径周围和之外扩展前沿。

## 2. 上下文输入

对于分配的分支：
- 策略文本和分支版本
- 最新执行或修正
- 最新批判
- 当前记忆银行（如果有）
- 最多 5 个之前的方案池输出
- 当前选择性假设包

对于其他活跃策略：
- 策略文本和分支版本
- 它们最新完成的方案池输出（仅最新）

不接收：
- 其他分支的修正、批判、历史或记忆银行

## 3. 输出格式

```json
{
  "strategy_id": "main-1",
  "solutions": [
    {
      "title": "方法标题",
      "content": "完整的候选方案内容",
      "confidence": 0.88,
      "internal_critique": "内部批判",
      "key_insights": "可选的关键见解"
    }
  ]
}
```

必须包含恰好 5 个条目。

## 4. BFS 组合构建策略

池组合应包含：
- 批判目标修复
- 不同表示或架构
- 对抗性伪造或反例
- 假设反转或替代目标
- 低置信度但高信息量的前沿候选

## 5. 反锚定

当相同结论通过局部修补存活时：
- 提供完全展开的机械替代品
- 展示替代结论在实际中如何成立
- 对于最优化任务：每个建设性候选都应尝试严格更好的值
