# REF98: Contextual Prompts 详解——主生成器

## 1. 核心协议

主生成器是一个迭代精炼系统中的自我修正器。

## 2. 关键指令

```typescript
// 强制性思维模式：
"原方案的结论可能完全错误。
 原方案的方法可能根本有缺陷。
 原推导可能包含微妙但致命错误。
 你必须愿意改变一切。"

// 禁止:
- 假设原答案"基本正确只需打磨"
- 试图通过修补来"拯救"原方案
- 在诊断证据面前维护原结论
- 需要根本修订时做最小更改
```

## 3. 必须输出结构

```
### Part 0: 进展追踪 (可选，非首次迭代)
记录: 哪些方法已尝试、成功/失败、学到什么

### Part 1: 回答问题 (可选，如果新方法不受问题影响可跳过)
Q1-Q5 回答

### Part 2: 方案选择
从方案池中选择或声明新方案

### Part 3: 修正后方案
完整方案正文
```

## 4. 循环检测

```typescript
// 检测信号：
- 多轮迭代中做出类似修正
- 批判持续指出同类问题
- 需要不断添加特例来处理异常
→ 方法根本性错误 → 必须采用根本不同的方法、框架、攻击角度
```

## 5. 方案池访问

- 方案池包含 12-15 个不同方案路径
- 每个方案使用不同的方法框架
- 主生成器可以选择、合并或无视方案池
- 不需要回答所有问题——如果新方法使旧问题无关，直接跳过
