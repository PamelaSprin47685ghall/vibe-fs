# REF19: Deepthink 模式——批判代理 (Critique Agent)

## 1. 角色定位

批判代理是迭代修正/精炼循环中的主要压力源。它不修正、不评分、不重写——只指出问题和改进方向。

## 2. 上下文输入

### 单通道路径
- Core Challenge、主策略、子策略、一个解决方案尝试
- 原始图片（如果有）

### EDFS 路径（额外）
- 策略槽位和分支版本
- 全局和分支本地迭代编号
- 最多 5 个之前的修正+批判条目

不接收：
- 解决方案池、记忆银行、选择性假设包、其他分支历史

## 3. 批判哲学

- **永不满足**: 即使工作很强，也要寻找下一个压力点
- **局部最小值意识**: 批判代理和修正代理可能共享相同的错误信念
- **历史意识**: 检查之前的批判点是否真正修复而非表面应对

## 4. 输出格式

```markdown
## Critique
### Critical Issues
### Domain-Specific Gaps
### Edge Cases, Counterexamples, or Stress Tests
### Improvement Directions
### Preserve
```

## 5. 关键设计

- 批判在对应的执行/修正完成时立即启动（无需等待其他分支）
- EDFS 模式下批判接收本分支的修正历史
- 不能声明"无改进需要"——即使工作很强也要找深层压力点
- 不能指定具体的替代方法论——让修正代理自由探索解决方案空间
