# REF23: Deepthink 模式——PQF (Post Quality Filter)

## 1. 角色定位

PQF 是分支维护机制，不是方案排名。它评估策略分支是否仍然值得探索。

## 2. 调用时机

- 每 5 个全局迭代触发一次
- 与记忆银行代理并行运行
- 策略按 2 个一组分组（`Math.ceil(N / 2)` 个 PQF 代理并行运行）

## 3. 上下文输入

- Core Challenge
- 选择的攻击性指令（balanced / very_aggressive）
- 所有当前活跃策略的文本（用于情境感知）
- 仅分配给该 PQF 组的 1-2 个策略的完整近期修正/批判历史

不接收：
- 方案池、假设包、记忆银行

## 4. 决策语义

| 决策 | 含义 |
|------|------|
| `keep` | 策略仍然有用，普通修正应继续 |
| `update` | 策略镜头本身失败，应在同一槽位启动新分支 |

**重要**: PQF 评估的是策略本身，不是某一轮修正中的局部缺陷。

## 5. 更新触发条件

- Balanced：仅当有根本性战略失败证据时
- Very Aggressive：更愿意替换持续概念弱点、域不匹配或低价值探索的分支

## 6. 策略替换流程

1. 所有 PQF 组完成后，聚合决策
2. 一个独立的**策略更新生成器**调用产生所有替换
3. 替换时：保留槽位 ID、递增分支版本、清除历史、清除选择性假设包
4. 替换分支执行新的首次执行和批判
