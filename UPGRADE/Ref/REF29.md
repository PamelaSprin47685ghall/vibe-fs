# REF29: Deepthink 模式——配置面板 (DeepthinkConfigPanel)

## 1. 配置项总览

配置面板是 React 组件，由 `DeepthinkConfigController` 驱动状态。

### 策略执行
| 配置 | 范围 | 说明 |
|------|------|------|
| 主策略数 | 1-10 (EDFS: 1-5) | 生成的并行策略分支数 |
| 子策略数 | 0/2-5 | EDFS 模式强制为 0 |

### 信息包
| 配置 | 范围 | 说明 |
|------|------|------|
| 假设启用 | on/off | 是否使用假设探针 |
| 假设数 | 0-6 | 生成的假设数量 |
| 注入模式 | parallel/strategy_aware/selective | EDFS 强制 selective |

### 方案精炼
| 配置 | 说明 |
|------|------|
| 启用精炼 | on/off | 关闭时无批判无修正 |
| 批判合成 | 开启时收集所有批判做全局诊断 |
| 完整方案上下文 | 所有原始方案和批判注入修正器 |
| EDFS | 复杂多轮迭代修正 |

### PQF
| 配置 | 说明 |
|------|------|
| Balanced | 仅当有根本性战略失败证据时更新 |
| Very Aggressive | 更激进地替换分支 |

## 2. 配置约束

| 设置 | 约束 |
|------|------|
| 精炼 | 开启批判、修正、合成、EDFS 的前提 |
| 主策略数（EDFS） | 1-5 |
| 子策略数（EDFS） | 强制 0 |
| PQF | EDFS 模式强制开启 |
| 注入模式（EDFS） | 强制 selective |
| 批判合成 | 需要精炼，EDFS 不可用 |
| 完整方案上下文 | 需要精炼，EDFS 不可用 |

## 3. Token 估算

配置面板内置 token 量大图表，根据策略数、假设数、深度估算总 token 消耗，支持线性/对数两种缩放。
