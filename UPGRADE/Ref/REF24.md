# REF24: Deepthink 模式——最终裁判 (Final Judge)

## 1. 角色定位

最终裁判是从所有完成的活跃候选中选择最佳方案的仲裁者。它与内部搜索机制有意分离。

## 2. 隔离原则

裁判只看到：
- Core Challenge
- 原始图片（如果有）
- 每个完成的活跃策略/子策略的候选方案
- 候选 ID、主策略 ID、子策略文本、最终方案文本

裁判看不到：
- 批判、记忆银行、假设包、方案池、PQF 决策、替换历史
- 其他代理的分数或评估

## 3. 评估标准（按重要性排序）

1. **功能正确性与鲁棒性**: 无编码错误、语法错误或逻辑缺陷
2. **完整性**: 完整、立即可用
3. **精炼深度与收敛成功**: 是否成功实现了分支的收敛目标
4. **用户价值**: 哪个方案最令用户满意

## 4. 输出格式

```json
{
  "best_solution_id": "main2-direct",
  "final_reasoning": "客观比较..."
}
```

## 5. 候选来源

- 对于有修正的分支：使用修正输出
- 对于无修正的分支：使用原始执行
- 已替换的分支归档供检查，但不作为最终候选

## 6. 防偏差设计

- 不按文风评分
- 不偏爱最长文本
- 不偏爱最抽象文本
- 不假设某些方法天生优越
