### 14. 死代码/过渡代码：`mkFileReadCapture` 禁用逻辑

**位置**：`src/Mux/Wrappers.fs:208-216`

```fsharp
let mkFileReadCapture (execute: obj -> JS.Promise<obj>) =
    let captured = HostFunctionCapture()
    captured.Set execute
    fun args ->
        promise {
            let! _ = captured.Execute args  // 执行但丢弃结果
            return box {| output = "disabled" |}   // 返回禁用
        }
```

**问题**：捕获文件读取执行函数后立即返回 `"disabled"`，已执行结果被丢弃。要么启用（不返回 disabled），要么删除。

**操作**：
- 若策略是禁止 Mux 的 file_read，直接不注册此 wrapper，删除 `mkFileReadCapture`
- 若策略是临时占位，加架构测试断言此禁用行为

**验收**：`mkFileReadCapture` 被删除或架构测试明确断言其行为。

## 不可行项

以下调查结论为**不需要**处理：

- **Kernel 纯度**：完全符合宝典要求，零 Dyn 渗透，架构测试自动守卫
- **类型边防**：Kernel 使用 `'raw` 参数正确传递宿主对象，不提前打开
- **编码器去重**：`encodePart` 虽有三处重复，但宿主差异（OMP entry vs Mux dynamic-tool vs Opencode tool part）使强制统一引入分支复杂度 > 收益
- **工具 codew 生成器**：14 个工具编码器各自 50-70 行，模板模式清晰，可维护，引入元编程得不偿失
- **`executor_wait` / `executor_abort`**：虽仅 OMP 使用，正确属于通用工具目录，不污染
