## 风险与注意事项

1. **循环依赖已修**：`BacklogSessionCodec` 上移至 `Shell`，`Omp/MagicTodo` 仅引用 `Shell.BacklogSessionCodec` + `Kernel.BacklogProjectionCore`，无 Omp→Opencode/Mux 交叉引用。
2. **`buildBacklogText` 类型已修**：`Kernel.BacklogProjectionCore.buildBacklogText` 保持 `BacklogEntry list → string`；`SessionCompacting` 传 `[]` 作 `userPrompts`，类型对齐。
3. **`src/Omp/` 引用约束**：已遵守——禁止 `open Wanxiangshu.Opencode` / `Wanxiangshu.Mux` / `engine/`。当前 Omp 仅引用 `Kernel` + `Shell` + `Omp.*`。
4. **`ToolResultEvent` 多键解析**：`getToolInput` / `getToolCallId` 三级 fallback 已覆盖 Pi 不同版本 payload 差异。
5. **nudge `hasActiveRunner`**：当前 Omp 恒 `false`。产品若需 executor 空闲提醒，需定义 Omp 侧检测源（对应 Opencode `NudgeRunner` 语义）。
6. **browser MCP**：Opencode 通过 `getClientFromPluginCtx` 注入 `stealth-browser-mcp`；Omp 无等价 client 入口，当前仅返回错误文本。
